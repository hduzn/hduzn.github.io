Github 图床使用教程

自己用的是 Github Pages + Hexo的博客，Markdown语言的静态网站，所以直接再用一个 Github的图床就比较方便。
前提当时是先注册一个 Github的账号。

1.Create a repository

创建一个仓库，专门准备用来放博客使用的图片。

repository 设置

名称只要是可用的都行，默认选择的是 Public，可以把 Add a README file勾选上。

设置好之后，最后点击【Create repository】。

2.创建 token

右上角头像下面的【Settings】->【Developer settings】->【Personal access tokens】->点击【Generate new token】。

  • Note：取个备注名
  • 勾选 repo

最后拉到页面最下面点击【Generate token】就创建好了token。

完成之后有一长串字母+数字的token，复制下来就行了。

3.用 PicGo工具上传图片

PicGo 官方下载：https://github.com/Molunerfinn/PicGo/releases

PicGo 官方介绍页面：https://picgo.github.io/PicGo-Doc/zh/guide/#%E5%90%AC%E8%AF%B4%E4%BD%A0%E4%B9%9F%E6%83%B3%E7%94%A8picgo

下载之后安装运行。

1).在PicGo设置中选择 Github图床

2).图床设置

  • 仓库名为：h***n/Picbed_Go （[github用户名]/[图床仓库名]）创建的时候上面有；
  • 分支名：点进 repository就能看到，之前默认都是 master；
  • Token：就是上面的token

3).上传区上传图片

把图片拖进去就能上传。

4).在相册里面可以看到上传的图片，图片下方第一个按钮就是复制，把地址复制出来直接就能在markdown中用。
