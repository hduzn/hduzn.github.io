在 Hexo 博客个性化定制中，如果有用豆瓣的，有自己标记的读书之类，可以直接导入生成一个豆瓣读书的页面，真的就很棒了。预览效果： 豆瓣读书

GitHub 上已经有人写好了这个模块：hexo-douban，我们只需要安装模块，然后配置一下就可以了。

一、安装

npm install hexo-douban --save

安装完后可以看到 Hexo\node_modules 目录下有 hexo-douban 目录。

二、配置

将下面的配置写入站点的配置文件 _config.yml 里(不是主题的配置文件)：

  • user: 你的豆瓣ID.打开豆瓣，登入账户，然后在右上角点击 “个人主页” ，这时候地址栏的URL大概是这样：”https://www.douban.com/people/xxxxxx/" ，其中的”xxxxxx”就是你的个人ID了。
  • builtin: 是否将生成页面的功能嵌入 hexo server 和 hexo generate 中，默认是false，另一可选项为true(1.x.x版本新增配置项)。
  • title: 该页面的标题。
  • quote: 写在页面开头的一段话,支持html语法。
  • timeout: 爬取数据的超时时间，默认是 10000ms ,如果在使用时发现报了超时的错(ETIMEOUT)可以把这个数据设置的大一点。

如果只想显示某一个页面(比如book)，那就把其他的配置项注释掉/删掉即可。

三、启用

现在用的是1.x.x的版本，使用 hexo douban 命令即可生成指定页面，如果不加参数，那么默认参数为-bgm。-bgm 代表的是 book、game、movie 三个参数，如果只需要其中的一部分就只带你想要的那些参数。

我们只需要在Git Bash中依次执行以下命令，就可以把豆瓣读书的页面添加到Hexo 博客中了。

hexo clean

hexo douban -b

（默认是-bgm，-b表示只加载book的。）
如果配置了builtin参数为true，那么除了可以使用 hexo douban 命令之外，hexo g 或 hexo s 也内嵌了生成页面的功能。

hexo g

hexo s

然后在本地打开 http://localhost:4000/books 页面查看，没有问题，再提交。

hexo deploy

注意：由于 hexo douban 的简写也是 hexo d，与 hexo deploy 的简写指令 hexo d 冲突，因此在进行二者部署的时候，只能都打全名而不能打简写形式。

五、个性化修改

执行完 hexo douban -b 命令后，其实可以看到就是在 Hexo\public 目录下生成了一个 books 目录，里面就一个index.html 文件。

所以，执行完后，我就直接把这个 books 目录直接拷到 Hexo\source 目录下，没有改动的话，也就不需要每次 hexo clean 后都重新生成一遍了。

于是，你可以通过修改 books 目录下的这个 index.html 文件，来修改一下这个读书的页面。

1、只保留【读过】选项。

默认生成的页面会，会有【在读】、【想读】、【读过】三个选项的内容，是对应豆瓣的。

我平常不标记【在读】，所以不想有这个。

【想读】标记的太多了，目前就有400多本，所以也不想显示这个。

2、继续编辑【推荐书目】。

因为读过的书都有200多本了，也比较杂，所以想把评分低的一些、不推荐的书就删了不显示了。

发现每本书的记录对应的div的class就是： <div class="hexo-douban-item">，那就简单了，把不需要的书的记录都删除就行 了。最后留下了100多本。

3、修改页面上颜色

修改颜色这些样式，需要修改 hexo-douban 中的css文件。所以这件事，其实你是应该可以先做的，修改完css后 再用 hexo douban -b 命令生成index.html文件，然后再修改index.html文件中的内容。

划重点：把 books\index.html 放到 Hexo\source 目录下后，只要这个读书页面没有改动，你就不用管啦。

不然生成在public目录下，每次hexo clean后就没有了，要重新生成一下，主要我还是书目比较多，就浪费时间了。你要是书目少的话，把配置中的 builtin参数为true 也行的。

PS.edge和手机浏览器会看不到图片，因为主题配置文件中默认开启了fancybox。这个设置需要设置为false，不然在点击图片进行跳转时会出现 The requested content cannot be loaded.Please try again later.错误
